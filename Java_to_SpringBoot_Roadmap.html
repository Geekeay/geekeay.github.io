Key reminders

  1. Learn Java 8 streams and lambdas before touching Spring Boot — every modern tutorial and codebase uses them. Nothing will read naturally until they do.
  2. Dependency Injection is the one concept that unlocks everything — every Spring annotation (@Service, @Autowired, @Component) is just this one idea. Understand it deeply before memorising annotations.
  3. What you remember from 2010 is mostly gone — no web.xml, no external Tomcat, no WAR deployments, no XML Spring config, no hand-written getters/setters. Don't relearn obsolete pain.
  4. Never expose JPA entities in your API — map to DTOs (records are perfect). The most common code-review comment in Spring projects.
  5. Start monolith, stay pragmatic — German interviewers penalise over-engineering as much as under-engineering. Microservices knowledge is for discussion, not your first project.
  6. This complements your AI/ML track — a FastAPI model-serving endpoint in Python is structurally identical to a Spring @RestController. The architecture thinking transfers 1:1.
