Key reminders

  1. Always state time and space complexity unprompted — right after your solution works, say "This is O(n) time, O(1) space because…". It is an explicit grading criterion in ML coding rounds.
  2. Solve in Python, not Java — ML interviewers expect Python fluency. Practising DSA in Python doubles as your Python training.
  3. Depth on Mediums beats grinding Hards — ML coding rounds sit at LC Medium. Master ~75–150 curated problems (Blind 75 → NeetCode 150) until you can re-solve them cold two days later.
  4. Expect ML-flavoured coding too — implementing k-means, logistic regression, or attention in pure NumPy is a real round at Google-style loops. Pure LeetCode prep alone is not enough.
  5. Talk while you code — narrate your approach, tradeoffs, and edge cases. Silent perfect code scores lower than spoken good code.
